Step 1: The Engine – Ollama (Your Local LLM Server)

Think of Ollama as Docker for LLMs: it downloads, runs, and serves models via a local REST API. It handles GPU acceleration automatically and is dead simple.

Installation (macOS / Linux / Windows WSL2)

# macOS / Linux (curl)
curl -fsSL https://ollama.com/install.sh | sh

# Windows: download from ollama.com/download (installer)
# After install, verify in terminal:
ollama --version

Once installed, pull a top-tier coding model. For most developers, deepseek-coder:6.7b-instruct offers the best balance of speed & intelligence.

# Pull the model (approx 3.8GB for 4-bit)
ollama pull deepseek-coder:6.7b-instruct-q4_K_M

# Or try llama3.1 8B
ollama pull llama3.1:8b

# Test it instantly
ollama run deepseek-coder:6.7b-instruct-q4_K_M "write a python function to reverse a linked list"

After pulling, Ollama runs a server on http://localhost:11434 automatically. This is the brain we'll connect to VS Code.

⚙️ Model recommendations for coding:
– DeepSeek-Coder-V2-Lite (16B) → best quality, needs 12GB RAM.
– CodeQwen1.5-7B-Chat → fast & excellent for Python/JS.
– Llama 3.1 8B Instruct → all-round solid.

Step 2: The Hands – Continue.dev VS Code Extension

Continue is the missing UI layer: it adds a copilot-like sidebar, autocomplete, inline edits, and a chat panel that talks directly to Ollama. It’s open-source, respects privacy, and works beautifully with local models.

Installation & Configuration

1. In VS Code, go to Extensions (Ctrl+Shift+X) and install "Continue" (by Continue Dev).
2. After install, open the Continue configuration file: Cmd+Shift+P → "Continue: Open Config File".
3. Replace the entire config.json with the block below: 

{
  "models": [
    {
      "title": "DeepSeek Coder Local",
      "provider": "ollama",
      "model": "deepseek-coder:6.7b-instruct-q4_K_M",
      "apiBase": "http://localhost:11434",
      "completionOptions": {
        "temperature": 0.2,
        "maxTokens": 2048
      }
    }
  ],
  "tabAutocompleteModel": {
    "title": "Local Autocomplete",
    "provider": "ollama",
    "model": "deepseek-coder:6.7b-instruct-q4_K_M",
    "apiBase": "http://localhost:11434"
  },
  "providers": [
    {
      "provider": "ollama",
      "apiBase": "http://localhost:11434"
    }
  ],
  "allowAnonymousTelemetry": false,
  "experimental": {
    "useContinueProxy": false
  }
}

Save the file, and voilà — Continue will connect to your local Ollama instance. You'll see a new Continue icon in the VS Code sidebar. Open it and start chatting with your local model.

Pro tip: To enable inline autocomplete (ghost text like Copilot), go to Continue settings → "Enable Tab Autocomplete". The model will suggest completions as you type, entirely offline.

Step 3: Context-Aware Coding – Index Your Codebase

The real magic happens when the AI understands your whole project structure. Continue has a built-in codebase indexing engine that creates embeddings for your repository. Here's how to activate it:

  • Open your project folder in VS Code.
  • In the Continue chat panel, type @codebase followed by your question. Example: @codebase where do we handle user authentication?
  • Continue will automatically index your files (once) and then retrieve relevant snippets before answering.
  • To manually rebuild the index: use the command palette → "Continue: Rebuild Codebase Index".

Indexing works locally with sentence-transformers/all-MiniLM-L6-v2 (downloaded once). This keeps everything private — embeddings never leave your machine. After indexing, you can ask complex architectural questions: "Refactor this API service to use async patterns" and it’ll reference your actual code.

🧠 Real dev workflow: Select a function → right-click → "Continue: Edit in chat" → prompt "add error handling and logging". The LLM sees the selected code and suggests modifications inline.

Beyond Basics: Roo Code, Custom Prompts & Tool Use

While Continue is our primary bridge, an emerging alternative is Roo Code (Cline fork) — it supports autonomous file editing and terminal execution. You can combine both: use Continue for chat/autocomplete, Roo for agentic workflows. To get started, just install "Roo Code" from marketplace and point it to the same Ollama endpoint.

For power users, create a custom system prompt inside Continue's config: add a "systemMessage" field to enforce style guides (e.g., "You are a senior Rust developer. Always include error handling and unit tests.").

Troubleshooting Common Hiccups

  • Ollama not reachable? Make sure the service is running: ollama serve (or it runs as daemon). Check http://localhost:11434 in browser.
  • Slow completions? Use a smaller quantized model or enable GPU acceleration (install CUDA for NVIDIA).
  • Autocomplete not showing? Inside Continue settings, set "Tab Autocomplete Model" and enable "Enable Tab Autocomplete".
  • High RAM usage? Set OLLAMA_NUM_PARALLEL=1 environment variable to limit concurrency.
